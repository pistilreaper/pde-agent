# 任务4：神经算子PDE智能体

## 赛事背景

在数据驱动的科学发现时代，Cell、Nature、Science（CNS）等顶级期刊不仅是前沿科学的殿堂，更已成为 AI for Science（AI4S）探索的核心阵地。从蛋白质结构的精准预测到复杂生命系统的模拟，AI 正在重塑基础科学的边界。然而，在传统科研流程中，AI 往往只作为静态辅助工具，其能力高度依赖人类科学家的手动调参。面对高维、多尺度、长时程的科学问题时，传统“人力主导”模式在处理大规模科学数据和复杂模型迭代方面已触及效率瓶颈。

自主科研 Agent 的兴起，预示着科研范式的下一次跃迁。本次挑战赛旨在探索 AI 在科学研究工作流中的自主能力：不仅让 Agent 自动化复现与优化前沿模型，更期望 Agent 能展现自主推理、设计、实验和迭代能力。

## Agent 核心任务：构建科研闭环能力

为了应对神经算子 PDE 预测任务，Agent 必须具备以下四个核心阶段的自动化能力：

- **文献解析与逻辑解构**：自主研读任务说明、PDEBench 数据规范、神经算子基线代码和相关文献，理清数学公式、数据流向、核心算法逻辑和科学问题本质。
- **瓶颈诊断与假设提出**：通过分析训练日志、验证指标和科学数据，识别精度、长时稳定性、泛化能力和计算效率上的瓶颈，并提出可验证的优化假设。
- **自主设计与代码演进**：根据诊断结果设计工程方案，如重构网络结构、引入条件化模块、设计损失函数或调整训练策略，并直接生成高质量代码。
- **实验验证与科学迭代**：自主运行实验、排查报错、分析失败原因、更新假设并记录完整科研轨迹。

## 赛题背景

偏微分方程（PDE）是刻画复杂动态物理系统的核心数学工具，广泛应用于气候模拟、流体力学、能源工程与材料科学等关键领域。传统数值方法具有严格物理一致性与高精度，但在高分辨率、多参数和长时间预测场景下计算代价极高。

神经算子（Neural Operator）通过学习函数空间到函数空间的映射，直接构建“初始条件 / 参数 / 边界条件 → 解场轨迹”的算子近似。一旦训练完成，原本依赖数值求解器反复迭代的预测过程可以通过一次或少量神经网络前向传播完成，实现显著加速。

从模型发展看，DeepONet 强调 Branch/Trunk 结构对非线性算子的通用近似能力，FNO 将积分核在 Fourier 空间参数化，PI-DeepONet / PINO 则将 PDE 残差和物理约束引入训练过程。尽管这些方法在效率和泛化方面取得进展，但模型设计、参数调优与验证流程仍高度依赖人工。比赛的核心问题是：能否构建面向 PDE 神经算子的科学智能体，使其自动完成“理解—改进—验证—复现”的科研闭环，并在效率与精度之间实现动态优化。

## 赛题介绍

本赛道定位为”AI4S 场景下的科研工作流 Agent 竞赛”：研究对象为神经算子（学习输入函数/系数/初边值 → 解场/时序轨迹 的算子映射），参赛者需开发 LLM Agent 自动完成“理解—改进—验证—复现”的科研闭环，而非仅提交一个训练好的模型权重。

本赛道的核心是从传统的“手动调参”向“自主科研”的范式跃迁。参赛者将不再扮演直接优化神经网络的“做题家”，而是要化身“全流程管理者”，开发一个能够独立接管科研工作流的 PDE Agent。

**构建什么样的 Agent？**
参赛者需要开发一个具备深度逻辑推理、架构演进与复杂代码工程能力的自动化智能体。该 Agent 需突破单纯的”对话交互”，深度整合工具调用（Tool-use）、代码执行沙盒与错误日志解析能力。

**如何运转科研闭环？**
Agent 被唤醒后，需在零人工干预的环境下，自主驱动完整的科学迭代闭环：

- 理解： 研读 PDEBench 数据集规范与主流神经算子（如 FNO、PI-DeepONet）的基线代码。
- 诊断与假设： 分析运行日志，敏锐识别基线模型在”精度”或”长时稳定性”上的物理与数学瓶颈，提出优化假说。
- 代码演进： 自主编写并修改高质量的 Python 代码（如重构网络拓扑、引入物理残差损失函数等）。
- 验证与反思： 运行干实验，像人类程序员一样排查报错（Debug），根据实验结果评估假说，调整方向并记录完整的科研日志。

比赛的目标并非单纯比拼谁能提供一个更优的模型权重，而是评测Agent 本身的“科研探索能力”​。最终，优秀的 Agent 将像经验丰富的科学家一样，通过严密的逻辑推导而非盲目算力试错，自主寻找到突破神经算子长时预测等业界难题的创新方案，并输出极具解释性的Agent 科研实验日志。

## 比赛任务

### Task 1：固定物理环境 1D Burgers 短窗预测

Task 1 要求使用固定粘性系数 $\nu = 0.001$ 的 1D Burgers 方程数据，给定测试轨迹的前 10 个时间步，预测完整 200 步轨迹中的后 190 步。

- `1D_Burgers_Sols_Nu0.001.hdf5`是 PDEBench 官方提供的 10000 个样本的 200 时间步长、1024 空间长度的数据集。可以考虑从该数据集中采样部分数据来微调官方checkpoint。
- 官方checkpoint: 1D_Burgers_Sols_Nu0.001_FNO.pt, 1D_Burgers_Sols_Nu0.001_Unet-PF-20.pt。
- 推荐优先路线是：基于官方 checkpoint 微调，或基于 PDEBench 大训练集采样训练/微调；`task1_val.hdf5` 默认仅作验证，不应直接因为参考实现如此而被默认当作训练集。
- 注意，由于模型在训练时设置的参数中 reduced_resolution_t=5, reduced_resolution=4，所以实际上模型是在 200/5=40 个时间步长，1024/4=256 个空间长度上进行验证，即：模型预测的 1 个时间步，相当于真实数据的 5 个时间步，空间长度同理。
- 验证集：`task1_val.hdf5` 中 `tensor`，shape `(100, 200, 256)`。

- 测试输入`task1_test.hdf5` 中 `tensor`，shape `(1000, 10, 256)`。
- 要求输出：`task1_pred.hdf5`，dataset `tensor`，shape `(1000, 200, 256)`。
- 一致性：输出前 10 步必须与测试输入一致，容差 `1e-3`。

### Task 2：多粘性系数 1D Burgers 泛化预测

Task 2 要求模型泛化到多种粘性系数 $\nu$。训练数据提供每条轨迹的 `nu`，测试集不提供 `nu`，因此模型必须从前 10 步初始观测中隐式或显式推断动力学参数。

- 测试输入：`task2_test.h5` 中 `tensor`，shape `(1000, 10, 256)`。
- 训练：`task2_part0_train.h5`、`task2_part1_train.h5`、`task2_part2_train.h5`，含 `nu`。
- 验证：`task2_val.h5`，含 `nu`。
- 输出：`task2_pred.hdf5`，dataset `tensor`，shape `(1000, 200, 256)`。
- 一致性：输出前 10 步必须与测试输入一致，容差 `1e-3`。
- 训练限制：必须从头训练，不得使用 Task 1 数据、Task 1 checkpoint、Task 1 fine-tuned 权重或任何外部预训练权重。
- 推理限制：推理时间必须在 120 秒内，否则该任务得 0 分。

### Task 3：Kuramoto-Sivashinsky 方程多参数神经算子预测（附加题）

Task 3 在 Burgers 任务基础上进一步提升难度，研究对象为多参数 Kuramoto-Sivashinsky（KS）方程：

$$
u_t + u \cdot u_x + \lambda_2 \cdot u_{xx} + u_{xxxx} = 0
$$

其中 $\lambda_2$ 为扩散/能量注入参数，$u_{xxxx}$ 项提供高波数耗散。二者与非线性对流项共同决定系统的时空混沌结构。KS 方程对初始条件高度敏感，逐点误差会随时间快速放大，因此 Task 3 同时考察短期预测精度和长时统计稳定性。

#### 任务定义

给定某一 KS 方程轨迹的前 20 个时间步（观测窗口 $t \in [0,9.5]$），预测完整 400 步轨迹（$t \in [0,199.5]$）。

- 输入：每个样本前 20 步，shape `(20, 256)`。
- 输出：完整 400 步轨迹，shape `(400, 256)`。
- 前 20 步必须与输入完全一致，后 380 步为预测结果。
- 训练集与验证集提供 $\lambda_2 \in [1.0,1.5]$；测试集不提供 $\lambda_2$。

#### 物理设置

| 参数 | 值 |
|---|---:|
| 空间格点数 | 256 |
| 存储时间步长 | $\Delta t_{stored}=0.5$ |
| 总时间步数 | 400（$t=0$ 到 $t=199.5$） |
| 参数范围 | $\lambda_2 \sim \mathrm{Uniform}[1.0,1.5]$ |

#### 数据文件

| 文件 | 样本数 | 时间步 | 是否含 $\lambda_2$ | 用途 |
|---|---:|---:|---|---|
| `task3_data_sample_submission/train_val_test_init/KS_train.hdf5` | 2000 | 400 | 是 | 训练 |
| `task3_data_sample_submission/train_val_test_init/KS_val.hdf5` | 100 | 400 | 是 | 验证 |
| `task3_data_sample_submission/train_val_test_init/KS_test.hdf5` | 100 | 20 | 否 | 测试输入 |

训练集 / 验证集 HDF5 结构：

```text
tensor        float32  (N, 400, 256)   # 完整轨迹
t-coordinate  float32  (400,)          # 时间坐标
x-coordinate  float32  (256,)          # 空间坐标
lambda2       float32  (N,)            # 每条轨迹的扩散系数
```

测试集 HDF5 结构：

```text
tensor        float32  (100, 20, 256) # 仅前 20 个时间步
t-coordinate  float32  (20,)
x-coordinate  float32  (256,)
```

#### Task 3 注意事项

- 训练总时长（含 Agent 思考时间）不计入精度评分，但须控制在 12 小时以内。
- 推理时间超过 120 秒则 Task 3 得分为 0。
- 推理时不得读取或假设测试集 $\lambda_2$，必须仅凭前 20 步观测输出预测。
- 至少提交 `task3_pred.hdf5`、`task3_time.csv`、`task3_logs.log` 三个文件才可参与 Task 3 评测。
- 禁止使用任何公开预训练权重或第三方 checkpoint。Task 3 模型必须完全基于 `KS_train.hdf5` 从头训练。

## 评分规则

系统首先通过日志文件进行规则检测。通过规则检测后，再按预测精度和时间计算得分。

### Task 1 评分（最高 150 分）

Task 1 总分 = 预测精度得分（最高 75）+ 训练耗时得分（最高 35）+ 推理耗时得分（最高 40）。

- 预测精度得分 = Task 1/2 通用分段预测得分 × 0.75。
- 训练耗时得分：≤60 分钟得 35 分，≤120 分钟得 25 分，≤300 分钟得 20 分，≤500 分钟得 10 分，>500 分钟得 0 分。
- 推理耗时得分：0 分钟得 40 分，0–2 分钟线性递减至 0 分，>2 分钟该任务得 0 分。

### Task 2 评分（最高 150 分）

Task 2 总分 = Task 1/2 通用分段预测得分 × 1.5。训练时间不计入评测，但总时长需控制在 12 小时内；推理超过 2 分钟则该任务总分为 0。

### Task 1 / Task 2 通用分段预测得分

评分仅针对 190 个预测时间步（去掉前 10 个初始条件）：

- 第 1 段（0–47 步，权重 25%）：`100 × exp(-20 × Rel-MSE)`。
- 第 2 段（47–95 步，权重 25%）：`100 × exp(-10 × Rel-MSE)`。
- 第 3 段（95–190 步，权重 50%）：`max(Lorentzian, Frechet)`。

其中 `Lorentzian = 100/(1+10×RMSE)`，`Frechet = 50×exp(-FD²)`。

### Task 3 评分（附加题，满分上限 350）

提交 Task 3 时，最终总分按以下两种方案分别计算并取较高者：

- **方案 A（Task 1 + Task 2 + Task 3）**：总分 = Task 1 得分 + Task 2 得分 + Task 3 分段预测得分 × 0.5，上限 350。
- **方案 B（Task 1 + Task 3）**：总分 = Task 1 得分 + Task 3 分段预测得分 × 2，上限 350。

Task 3 分段预测得分仅针对 380 个预测步（去掉前 20 个观测步）：

| 段 | 完整轨迹步范围 | 物理时间 | 权重 | 评分公式 |
|---|---:|---:|---:|---|
| 第 1 段 | 20–49 | $t \in [10,24.5]$ | 25% | `100 × exp(-20 × Rel-MSE1)` |
| 第 2 段 | 50–199 | $t \in [25,99.5]$ | 25% | `100 × exp(-10 × Rel-MSE2)` |
| 第 3 段 | 200–399 | $t \in [100,199.5]$ | 50% | `max(Lorentzian, Frechet)` |

第 3 段中 `Lorentzian = 100/(1+10×RMSE3)`，`Frechet = 50×exp(-FD²)`。Task 3 分段预测总分为 `0.25×s1 + 0.25×s2 + 0.50×s3`，范围 0–100。
