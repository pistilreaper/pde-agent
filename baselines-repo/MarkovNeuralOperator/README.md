# Markov Neural Operator (MNO) — baseline reference pack

This pack is intended for a CozyPDE / PDEAgent run as a **method-reading reference**, not as final submission code.

## What the upstream method does

MNO is directly relevant to dissipative chaotic systems. The upstream README states that the work evaluates Lorenz-63, 1D Kuramoto–Sivashinsky, and 2D Navier–Stokes/Kolmogorov flow, and learns a Markovian solution operator that captures short-time trajectories and long-time statistics.

Upstream repository: https://github.com/neuraloperator/markov_neural_operator  
Paper / method reference: https://arxiv.org/abs/2106.06898

## Heuristics for the Agent

- Treat Task 3 as learning a Markovian time-stepper or chunked flow map, not necessarily a single direct 20→400 map.
- Add long-rollout validation during training; short-horizon MSE alone is insufficient for chaotic KS.
- Use Sobolev/Hk or spectral-gradient losses to penalize derivative/spectrum mismatch.
- Use dissipativity-inspired regularization: keep rollout energy/norm bounded rather than letting recursive predictions explode.
- For Task 3, adapt the idea only; do not use MNO pretrained weights or external generated trajectories.

## For Task 3

Task 3 is a multi-parameter Kuramoto–Sivashinsky prediction task with an unknown test-time parameter. It rewards short-horizon accuracy but also long-time statistical stability. The Agent should therefore extract ideas about long rollout, spectral stability, parameter inference, and safe data augmentation, while still training only from official Task 3 data.
